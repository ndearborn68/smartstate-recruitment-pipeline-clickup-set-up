Multi-Channel Outreach Flow

Three outreach channels feed into one ClickUp pipeline. Every contact is logged to prevent duplicates.

Channel 1: Instantly (Email Campaigns)

1
Load leads into Instantly with full data
Push leads with name, email, LinkedIn, phone into Instantly campaign. All this data gets captured.
2
Campaign launches → Claude creates ClickUp tasks
When campaign sends, each lead gets a task in ClickUp with status "Outreach Sent", channel "Instantly", and campaign name. Pushed via MCP.
3
Reply detected → Update task to "Replied"
Instantly webhook or you tell Claude → task moves to "Replied" with date. Resume scanning kicks in if they send one.

Channel 2: Heyreach (LinkedIn Outreach)

1
Heyreach sequence launches
LinkedIn connection requests / messages go out via Heyreach automation.
2
Heyreach API → ClickUp task created
Connect Heyreach API to ClickUp. Each contacted lead becomes a task with channel "Heyreach" and sequence name.
3
LinkedIn reply → Task updated to "Replied"
Heyreach detects reply → webhook or you tell Claude → task moves to "Replied".

Channel 3: LinkedIn Recruiter (Manual InMails) NO API

1
You send InMails via LinkedIn Recruiter
Manual outreach to passive candidates. No API to auto-sync.
2
Log contacts via Claude (batch or one-by-one)
After your session: tell Claude the names + LinkedIn URLs. Claude creates tasks in ClickUp with channel "LI Recruiter". See "LI Recruiter Fix" tab for details.
3
Reply? Tell Claude → update to "Replied"
When someone replies to your InMail, tell Claude to update their status.

Channel 4: Inbound (Email Replies with Resume)

1
Candidate sends resume via email
Email automation scans for resume attachment, extracts name + email.
2
Resume attached to existing task (or new task created)
If the person already has a task (from Instantly/Heyreach/LI Recruiter), the resume is attached to it. Otherwise a new "Inbound" task is created.

Deduplication Logic

ClickUp is the central dedup hub. Before ANY outreach, check if the person already exists.

How Dedup Works

Every candidate is matched by LinkedIn URL (primary) or email address (secondary). These are unique identifiers.

Scenario 1: Pre-Campaign Check
Before loading leads into Instantly or Heyreach, tell Claude: "Check if these people are already in ClickUp for Sr. Software Engineer: [paste list]." Claude searches by LinkedIn URL and email, flags duplicates, and tells you who's safe to add.
Scenario 2: Cross-Channel Conflict
You're about to InMail someone on LinkedIn Recruiter. Ask Claude: "Is sarah.chen@email.com already being contacted?" Claude checks all channels and tells you: "Yes — she's in Instantly campaign SmartState-SWE-Q1, emailed Mar 4. Status: Outreach Sent."
Scenario 3: Same Person, Different Channel
Someone already contacted via Heyreach applies inbound via email. Instead of creating a duplicate, the resume attachment gets added to their existing task, and you're notified.
Scenario 4: Bulk Dedup Before Campaign Launch
You're loading 150 leads into Instantly. Export your ClickUp candidate list first. Claude cross-references and removes anyone already contacted via Heyreach or LI Recruiter from your Instantly upload.

Dedup Commands:

"Is linkedin.com/in/sarahchen already in any SmartState job?"
"Check these 20 LinkedIn URLs against ClickUp before I load them into Instantly"
"Show me all candidates contacted through multiple channels"
"Export everyone in Sr. Software Engineer so I can dedup against my Heyreach list"

LinkedIn Recruiter Solution

LinkedIn Recruiter has no API. Here are three ways to keep ClickUp in sync — I recommend using them together.

Recommended Approach: Three-Layer Solution

Each layer handles a different moment in your workflow
1
Batch Log via Claude After Each Session PRIMARY
After an LI Recruiter session, paste the list of people you InMailed. Example: "I just InMailed these 5 people from LinkedIn Recruiter for Sr. Software Engineer:" then paste names + LinkedIn URLs. Claude bulk-creates all 5 tasks in ClickUp with channel "LI Recruiter" and today's date. Takes 30 seconds.
2
Pre-Outreach Dedup Check ALWAYS DO THIS
Before sending any InMail, tell Claude: "I'm about to InMail these people — check if any are already in ClickUp." Claude searches by LinkedIn URL across ALL channels and flags conflicts. This prevents double-contacting someone already in an Instantly or Heyreach sequence.
3
Weekly Sync via LinkedIn Recruiter Export
LinkedIn Recruiter lets you export project/pipeline lists as CSV. Once a week, export your "Sent InMails" list. Give it to Claude and say "Sync this with ClickUp — flag anyone missing." Claude creates any tasks that weren't logged in real-time.
